# 07 Authentication

## 개요

이 문서는 Flask에서 사용자 인증의 기본 개념인 회원가입, 로그인, 로그아웃, 세션 사용을 설명합니다.

## 인증이란?

인증(authentication)은 사용자가 누구인지 확인하는 과정입니다.

대표적인 기능:

- 회원가입
- 로그인
- 로그아웃

## 세션

로그인 상태를 유지하려면 세션(session)을 사용합니다.

```python
session['username'] = user['username']
```

세션에 값을 저장하면 다음 요청에서도 로그인 정보를 유지할 수 있습니다.

## SECRET_KEY

Flask에서 세션을 사용하려면 비밀 키가 필요합니다.

```python
app.config['SECRET_KEY'] = 'simple-secret-key'
```

이 값은 실제 서비스에서는 더 안전한 값으로 관리해야 합니다.

## 회원가입

회원가입에서는 보통 다음을 처리합니다.

- 사용자명 입력
- 이메일 입력
- 비밀번호 입력
- 비밀번호 확인
- 중복 사용자 확인

## 로그인

로그인에서는 입력한 사용자명과 비밀번호를 데이터베이스와 비교합니다.

정상적으로 일치하면 세션에 사용자 정보를 저장합니다.

## 로그아웃

로그아웃은 세션을 비우는 방식으로 처리할 수 있습니다.

```python
session.clear()
```

## 보호된 페이지

로그인한 사용자만 접근해야 하는 페이지가 있을 수 있습니다.

예:

- 대시보드
- 마이페이지
- 관리자 화면

이런 경우 세션에 로그인 정보가 있는지 확인해 접근을 제어합니다.

## 주의할 점

인증 기능에서는 비밀번호를 평문으로 저장하면 안 됩니다.  
학습용 예제에서는 단순하게 보일 수 있지만, 실제 서비스에서는 반드시 암호화가 필요합니다.

## 핵심 정리

- 인증은 사용자 신원을 확인하는 기능입니다.
- 로그인 상태 유지는 세션으로 처리합니다.
- 회원가입, 로그인, 로그아웃이 기본 흐름입니다.
- 보호된 페이지는 세션 확인을 통해 접근을 제어합니다.
