Flask 개요
==========

1. Flask란?
-----------
Flask는 Python으로 웹 애플리케이션을 만들기 위한 가벼운 웹 프레임워크입니다.

브라우저의 요청을 받아 처리하고, 그 결과를 HTML이나 JSON으로 응답하는 웹 프로그램을 만들 때 사용합니다.
Flask는 마이크로 프레임워크로 분류되며, 필요한 기능부터 작게 시작한 뒤 점진적으로 확장하기 좋은 구조를 가집니다.

2. Flask로 할 수 있는 것
------------------------
- 페이지 라우팅
- HTML 템플릿 렌더링
- 폼 처리
- 데이터베이스 연결
- 로그인과 세션 처리
- 파일 업로드
- 게시판 같은 CRUD 기능
- JSON API 응답

3. Flask의 특징
---------------
- 구조가 비교적 단순하고 이해하기 쉽다
- 필요한 기능부터 작게 시작할 수 있다
- Python 라이브러리와 함께 사용하기 좋다
- 라우팅, 템플릿, 요청/응답 흐름을 배우기에 적합하다
- 데이터베이스, 인증, 파일 업로드 같은 기능을 점진적으로 추가할 수 있다

4. Flask의 장점
---------------
1) 배우기 쉽다
   - 코드가 비교적 짧고 흐름이 단순하다
   - 웹 개발의 기본 구조를 이해하기 좋다

2) 자유도가 높다
   - 프로젝트 구조를 비교적 자유롭게 설계할 수 있다
   - 필요한 기능을 선택해서 붙일 수 있다

3) Python과 잘 어울린다
   - Python 문법과 자연스럽게 연결된다
   - 다른 Python 라이브러리와 함께 사용하기 좋다

4) 작은 프로젝트부터 시작하기 좋다
   - 수업용 예제
   - 개인 프로젝트
   - 프로토타입 제작

5. Flask의 단점
---------------
1) 많은 기능이 처음부터 자동으로 들어 있지는 않다
   - 필요한 기능을 직접 선택하고 구성해야 할 수 있다

2) 구조를 스스로 잡아야 하는 경우가 많다
   - 프로젝트가 커질수록 설계가 중요해진다

3) 대규모 프로젝트에서는 추가 설계가 필요하다
   - 인증, 권한, 데이터베이스 구조, 파일 관리 등을 더 체계적으로 설계해야 한다

6. Flask는 언제 적합한가
------------------------
- Python 웹 개발을 처음 배울 때
- 웹의 요청-응답 구조를 이해하고 싶을 때
- 작고 빠르게 동작하는 웹앱을 만들고 싶을 때
- 수업용 예제나 실습 프로젝트를 구성할 때

7. Flask 요청-응답 흐름
-----------------------
1. 사용자가 브라우저에서 URL에 접속한다
2. Flask가 URL과 연결된 함수를 찾는다
3. 함수에서 데이터를 처리한다
4. HTML 또는 JSON 응답을 만든다
5. 브라우저가 결과를 화면에 표시한다

8. 이 폴더의 학습 흐름
----------------------
- 00_Hello_Flask.py: 첫 앱 생성과 기본 라우팅
- 01_Routing.py: 라우팅 확장 기능
- 02_Static_Files.py: 정적 파일 처리
- 03_Templates.py: Jinja 템플릿
- 04_Forms.py: 폼 처리
- 05_Database.py: SQLite 데이터베이스 연결
- 06_CRUD.py: 생성, 조회, 수정, 삭제
- 07_Authentication.py: 로그인과 세션
- 08_File_Upload.py: 파일 업로드
- 09_Board.py: 게시판 형태의 종합 예제

9. 설치
-------
pip install flask

10. 실행 예시
------------
python 00_Hello_Flask.py

브라우저에서 안내된 주소로 접속해 결과를 확인합니다.
