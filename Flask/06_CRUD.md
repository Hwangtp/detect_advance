# 06 CRUD

## 개요

이 문서는 웹 애플리케이션에서 가장 기본이 되는 CRUD 기능을 설명합니다.

CRUD는 다음 네 가지 동작을 뜻합니다.

- Create: 생성
- Read: 읽기
- Update: 수정
- Delete: 삭제

## Create

새로운 게시글을 작성해 데이터베이스에 저장하는 기능입니다.

폼에서 제목, 내용, 작성자를 입력받고, `INSERT` 쿼리로 저장합니다.

## Read

저장된 게시글을 조회하는 기능입니다.

이 파일에서는

- 게시글 목록
- 게시글 상세 보기
- 검색
- 페이징

까지 포함되어 있습니다.

## Update

기존 게시글을 수정하는 기능입니다.

기존 데이터를 불러와 폼에 채우고, 수정 후 `UPDATE`로 다시 저장합니다.

## Delete

특정 게시글을 삭제하는 기능입니다.

보통 삭제 후에는 목록 페이지로 다시 이동시킵니다.

## redirect와 상세 페이지 이동

게시글 생성이나 수정 후에는 보통 해당 게시글 상세 페이지로 이동합니다.

```python
return redirect(url_for('read_article', article_id=article_id))
```

## 검색과 페이징

CRUD는 단순히 생성·수정·삭제만이 아니라,

- 많은 데이터 중 필요한 내용을 찾고
- 여러 페이지로 나누어 보여 주는 기능

과도 자연스럽게 연결됩니다.

## 추가 기능

이 파일에는 복제 기능과 통계 기능도 포함되어 있습니다.

- 게시글 복제
- 작성자별 통계
- 전체 게시글 수 집계

## 핵심 정리

- CRUD는 웹 애플리케이션의 가장 기본적인 데이터 처리 흐름입니다.
- 폼, 데이터베이스, 라우팅, 템플릿이 함께 동작합니다.
- 목록, 상세, 수정, 삭제 흐름을 구현하면 게시판 구조의 기초를 이해할 수 있습니다.
