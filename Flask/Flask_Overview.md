# Flask Overview

## Flask란?

Flask는 Python으로 웹 애플리케이션을 만들기 위한 가벼운 웹 프레임워크입니다.  
브라우저의 요청을 받아 처리하고, 그 결과를 HTML이나 JSON 형태로 응답하는 웹 프로그램을 만들 때 자주 사용됩니다.

Flask는 "마이크로 프레임워크"라고 불리는데, 이것은 기능이 부족하다는 뜻이 아니라  
처음부터 모든 기능을 강제로 넣지 않고 필요한 기능을 차례대로 추가할 수 있다는 뜻에 가깝습니다.

즉, Flask는 웹 개발의 핵심 구조를 비교적 단순한 형태로 보여 주기 때문에  
Python 기반 웹 개발 입문용으로 많이 사용됩니다.

## Flask로 할 수 있는 것

Flask를 사용하면 다음과 같은 웹 기능을 만들 수 있습니다.

- 페이지 라우팅
- HTML 템플릿 렌더링
- 폼 처리
- 데이터베이스 연결
- 로그인과 세션 처리
- 파일 업로드
- 게시판 같은 CRUD 기능
- JSON API 응답

단순한 실습용 사이트부터 일정 규모의 웹 애플리케이션까지 확장할 수 있습니다.

## Flask의 특징

- 구조가 비교적 단순하고 이해하기 쉽습니다.
- 필요한 기능부터 작게 시작할 수 있습니다.
- 라우팅, 템플릿, 요청/응답 흐름을 배우기에 적합합니다.
- Python 라이브러리와 함께 사용하기 좋습니다.
- 데이터베이스, 인증, 파일 업로드 같은 기능을 점진적으로 추가할 수 있습니다.

## Flask의 장점

### 1. 배우기 쉽다

Flask는 코드가 비교적 짧고 흐름이 단순해서 웹 개발의 기본 구조를 이해하기 좋습니다.

### 2. 자유도가 높다

프로젝트 구조를 비교적 자유롭게 설계할 수 있습니다.  
필요한 기능을 직접 선택해서 붙일 수 있다는 점도 장점입니다.

### 3. Python과 잘 어울린다

기존 Python 문법과 자연스럽게 연결되며, 다른 Python 라이브러리와 함께 사용하기 좋습니다.

### 4. 작은 프로젝트부터 시작하기 좋다

간단한 예제, 수업용 프로젝트, 개인 웹앱, 프로토타입 제작에 잘 맞습니다.

## Flask의 단점

### 1. 처음부터 많은 기능이 자동으로 들어 있지는 않다

장점이기도 하지만, 큰 프로젝트에서는 필요한 기능을 직접 선택하고 구성해야 할 수 있습니다.

### 2. 구조를 스스로 잡아야 하는 경우가 많다

자유도가 높은 대신, 프로젝트가 커질수록 구조를 어떻게 나눌지 고민이 필요합니다.

### 3. 대규모 프로젝트에서는 추가 설계가 중요하다

기능이 많아질수록 인증, 권한, 데이터베이스 구조, 파일 관리 등을 더 체계적으로 설계해야 합니다.

## Flask는 언제 적합한가

Flask는 다음과 같은 상황에 잘 어울립니다.

- Python 웹 개발을 처음 배울 때
- 웹의 요청-응답 구조를 이해하고 싶을 때
- 작고 빠르게 동작하는 웹앱을 만들고 싶을 때
- 수업용 예제나 실습 프로젝트를 구성할 때

즉, Flask는 "웹 개발 전체 흐름을 배우는 입문 프레임워크"로 매우 적합합니다.

## 웹 프레임워크로서 Flask의 흐름

Flask 애플리케이션은 보통 다음 순서로 동작합니다.

1. 사용자가 브라우저에서 URL에 접속합니다.
2. Flask가 해당 URL과 연결된 함수를 찾습니다.
3. 함수에서 데이터를 처리합니다.
4. HTML 또는 JSON 형태의 응답을 만듭니다.
5. 브라우저가 결과를 화면에 표시합니다.

## 이 폴더의 학습 흐름

이 폴더는 Flask의 기초부터 게시판 형태의 응용 예제까지 순서대로 구성되어 있습니다.

- `00_Hello_Flask.py`: 첫 Flask 앱과 기본 라우팅
- `01_Routing.py`: 라우팅 확장 기능
- `02_Static_Files.py`: CSS, JavaScript, 이미지 같은 정적 파일
- `03_Templates.py`: Jinja 템플릿
- `04_Forms.py`: 폼 처리
- `05_Database.py`: SQLite 데이터베이스 연결
- `06_CRUD.py`: 생성, 조회, 수정, 삭제
- `07_Authentication.py`: 로그인과 세션
- `08_File_Upload.py`: 파일 업로드
- `09_Board.py`: 게시판 형태의 종합 예제

학습 흐름은 다음과 같이 이어집니다.

- `Overview`: Flask가 무엇인지 이해
- `00`: 첫 앱 생성과 가장 기본적인 라우팅
- `01` ~ `04`: 라우팅, 정적 파일, 템플릿, 폼 처리
- `05` ~ `09`: 데이터베이스, CRUD, 인증, 업로드, 게시판

## 필요한 라이브러리

```bash
pip install flask
```

## 기본 실행 방법

각 파일은 독립적으로 실행할 수 있습니다.

```bash
python 00_Hello_Flask.py
```

그 뒤 브라우저에서 안내된 주소로 접속해 결과를 확인합니다.

## 함께 알아두면 좋은 폴더

Flask 프로젝트에서는 보통 다음 구조가 함께 등장합니다.

- `templates/`: HTML 템플릿
- `static/`: CSS, JavaScript, 이미지
- `uploads/`: 업로드 파일
- `*.db`: SQLite 데이터베이스 파일

## 핵심 정리

Flask는 Python 기반 웹 개발의 핵심 흐름을 배우기에 적합한 가벼운 웹 프레임워크입니다.  
장점은 단순함과 유연성이며, 단점은 큰 프로젝트에서 구조를 직접 더 많이 설계해야 한다는 점입니다.  
이 폴더는 Flask의 개념 이해에서 시작해 실제 웹 기능 구현으로 자연스럽게 이어지도록 구성되어 있습니다.
