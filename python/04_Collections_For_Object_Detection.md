# 자료, 자료처리, 자료구조 그리고 컬렉션의 중요성

파이썬의 컬렉션 자료구조를 배울 때 가장 먼저 보게 되는 것은 `list`, `tuple`, `dict`, `set` 같은 문법입니다.  
하지만 자료구조는 단순한 문법 분류가 아니라, 데이터를 어떻게 정리하고 다룰지를 결정하는 핵심 개념입니다.

자료구조를 이해하려면 먼저 세 가지 개념을 구분할 필요가 있습니다.

- 자료
- 자료처리
- 자료구조

이 세 가지가 연결되면 컬렉션은 단순한 기호가 아니라 프로그래밍에서 매우 중요한 도구로 보이게 됩니다.

---

## 1. 자료란 무엇인가

자료는 프로그램이 다루는 모든 값입니다.  
숫자, 문자열, 이름, 점수, 날짜, 좌표, 상품 정보처럼 프로그램 안에서 저장하고 사용할 수 있는 값은 모두 자료입니다.

```python
age = 20
name = "Kim"
score = 95
```

위 코드에서 `20`, `"Kim"`, `95`는 각각 하나의 자료입니다.

- `20`은 나이 자료
- `"Kim"`은 이름 자료
- `95`는 점수 자료

즉, 자료는 프로그램이 기억하고 처리하는 대상입니다.

---

## 2. 자료처리란 무엇인가

자료처리는 저장된 자료를 이용해 어떤 작업을 수행하는 것입니다.

예를 들어 다음과 같은 작업은 모두 자료처리입니다.

- 더하기
- 비교하기
- 정렬하기
- 검색하기
- 출력하기
- 저장하기
- 수정하기
- 삭제하기

```python
scores = [88, 92, 76, 95, 83]
```

이 자료에 대해 할 수 있는 처리도 많습니다.

- 평균 구하기
- 최고점 찾기
- 오름차순 정렬하기
- 90점 이상 값만 찾기

자료가 재료라면 자료처리는 그 재료를 가지고 하는 작업입니다.

---

## 3. 자료구조란 무엇인가

자료구조는 자료를 저장하고 조직하는 방법입니다.  
같은 자료라도 어떤 구조로 묶어 두느냐에 따라 처리 방식, 코드의 가독성, 수정 편의성이 달라집니다.

```python
student1 = "Kim"
student2 = "Lee"
student3 = "Park"
```

이렇게도 저장할 수 있지만, 자료가 많아질수록 관리가 어려워집니다.

```python
students = ["Kim", "Lee", "Park"]
```

이처럼 여러 자료를 하나의 구조로 묶으면 반복, 검색, 추가, 삭제 같은 처리가 쉬워집니다.  
바로 이런 정리 방식이 자료구조입니다.

---

## 4. 왜 자료구조가 중요한가

프로그래밍에서는 자료를 어떻게 저장했는지가 자료를 어떻게 처리할 수 있는지를 크게 좌우합니다.

예를 들어 같은 학생 정보도 아래 두 방식은 차이가 큽니다.

```python
student = ["Kim", 20, 95]
```

```python
student = {"name": "Kim", "age": 20, "score": 95}
```

첫 번째 방식은 저장은 가능하지만 의미가 불분명합니다.

- `student[0]`은 무엇인가
- `student[1]`은 나이인가 점수인가

반면 두 번째 방식은 이름표가 있어서 의미가 분명합니다.

- `student["name"]`
- `student["age"]`
- `student["score"]`

자료구조는 단순한 저장 방식이 아니라, 데이터를 이해하고 활용하는 방식을 결정하는 설계입니다.

---

## 5. 자료구조를 잘 이해하면 좋은 점

자료구조를 잘 이해하면 프로그램 전체가 더 안정적이고 읽기 쉬워집니다.

### 1) 코드를 읽기 쉬워진다

좋은 자료구조는 코드에 의미를 드러냅니다.  
무엇이 저장되어 있는지 파악하기가 쉬워집니다.

### 2) 자료처리가 쉬워진다

정렬, 검색, 반복, 추가, 삭제 같은 작업은 자료구조에 따라 쉬워지거나 복잡해질 수 있습니다.

### 3) 실수를 줄일 수 있다

의미가 불분명한 구조는 인덱스를 헷갈리게 하고 버그를 만들기 쉽습니다.

### 4) 기능 확장이 쉬워진다

처음에는 작은 프로그램이라도 나중에는 더 많은 정보가 붙습니다.  
자료구조가 잘 잡혀 있으면 수정과 확장이 쉬워집니다.

### 5) 문제 해결력이 좋아진다

프로그래밍은 단순히 문법을 외우는 일이 아니라, 데이터를 어떤 구조로 담아야 다루기 좋은지 판단하는 과정이기도 합니다.

---

## 6. 컬렉션 자료구조가 필요한 이유

현실의 데이터는 대부분 한 개가 아니라 여러 개입니다.

- 학생 여러 명
- 점수 여러 개
- 상품 여러 개
- 단어 여러 개
- 좌표 여러 개

이처럼 여러 자료를 한 번에 다루기 위해 컬렉션 자료구조가 필요합니다.

파이썬의 대표적인 컬렉션 자료구조는 다음과 같습니다.

- 리스트 `list`
- 튜플 `tuple`
- 딕셔너리 `dict`
- 셋 `set`

이들은 모두 여러 자료를 묶는 구조이지만, 목적과 특징이 다릅니다.

---

## 7. 리스트는 여러 자료를 순서대로 모아 두는 구조

리스트는 가장 기본적인 컬렉션 자료구조입니다.

```python
scores = [88, 92, 76, 95, 83]
```

리스트의 특징은 다음과 같습니다.

- 순서가 있다
- 수정할 수 있다
- 같은 값이 여러 번 들어갈 수 있다
- 반복 처리에 적합하다

리스트는 학생 명단, 점수 목록, 주문 목록처럼 순서대로 나열된 여러 자료를 다룰 때 자연스럽습니다.

---

## 8. 튜플은 고정된 자료 묶음에 적합하다

튜플도 여러 값을 묶는 구조이지만, 한 번 만들면 바꾸기 어렵다는 특징이 있습니다.

```python
point = (10, 20)
birth = (2000, 5, 12)
```

튜플은 아래와 같은 자료에 잘 어울립니다.

- 좌표
- 날짜
- 크기
- 색상

즉, 값들의 순서와 의미는 중요하지만 자주 바뀌지 않는 자료에 적합합니다.

---

## 9. 딕셔너리는 이름표가 붙은 자료구조다

딕셔너리는 `key`와 `value`의 쌍으로 자료를 저장합니다.

```python
student = {
    "name": "Kim",
    "age": 20,
    "score": 95
}
```

딕셔너리의 장점은 자료의 의미가 분명하다는 점입니다.

- `"name"` 아래 이름 저장
- `"age"` 아래 나이 저장
- `"score"` 아래 점수 저장

정보가 복잡할수록 딕셔너리는 더 유용해집니다.  
인덱스 번호를 기억하는 대신 이름으로 접근할 수 있기 때문입니다.

---

## 10. 셋은 중복을 허용하지 않는 구조다

셋은 중복된 값을 제거하거나 포함 여부를 빠르게 확인할 때 유용합니다.

```python
numbers = [1, 2, 2, 3, 3, 3, 4]
unique_numbers = set(numbers)
```

결과는 다음과 같습니다.

```python
{1, 2, 3, 4}
```

셋은 순서보다 중복 제거와 포함 여부 확인이 더 중요한 상황에서 강점을 가집니다.

---

## 11. 자료구조를 잘 고르면 자료처리가 쉬워진다

자료구조의 중요성은 어떤 자료가 들어 있느냐보다 그 자료를 나중에 어떻게 처리할 것인가와 연결됩니다.

예를 들어 이름표가 필요한 정보라면 딕셔너리가 적합하고,  
순서대로 반복 처리할 자료라면 리스트가 적합하고,  
중복 제거가 중요하다면 셋이 적합합니다.

즉, 자료구조 선택은 저장의 문제가 아니라 처리 전략의 문제이기도 합니다.

> 자료구조는 데이터를 담는 방법이고,  
> 자료처리는 그 데이터를 사용하는 방법이다.  
> 데이터를 잘 담아야 잘 사용할 수 있다.

---

## 12. 자료구조를 잘못 선택하면 생기는 문제

자료구조를 적절히 선택하지 않으면 다음과 같은 문제가 생길 수 있습니다.

- 코드가 읽기 어려워진다
- 데이터의 의미가 흐려진다
- 실수가 늘어난다
- 기능 추가가 어려워진다
- 처리 로직이 복잡해진다

반대로 자료구조가 잘 잡혀 있으면 코드는 더 짧고 분명해지고, 수정도 쉬워집니다.

---

## 13. 핵심 정리

프로그래밍은 결국 자료를 다루는 일입니다.  
자료가 많아질수록 정리 방식이 필요하고, 그 정리 방식이 바로 자료구조입니다.

- 자료구조가 좋아야 자료처리가 쉬워진다
- 자료구조가 분명해야 코드가 읽기 쉬워진다
- 자료구조를 잘 이해할수록 문제를 더 명확하게 다룰 수 있다

한 문장으로 정리하면 다음과 같습니다.

**자료구조를 배운다는 것은 데이터를 잘 저장하는 법만 배우는 것이 아니라, 데이터를 잘 다루는 법을 배우는 것이다.**

---

## 14. 마무리

컬렉션 자료구조는 단순히 여러 값을 넣는 문법이 아닙니다.

- 리스트는 여러 값을 순서대로 모은다
- 튜플은 고정된 의미의 값을 묶는다
- 딕셔너리는 이름표와 함께 저장한다
- 셋은 중복을 제거한다

이 차이는 단순한 모양의 차이가 아니라 자료를 처리하는 방식의 차이로 이어집니다.  
따라서 자료구조를 이해하는 일은 프로그래밍의 문법을 넘어서, 문제를 더 명확하게 보고 데이터를 더 효과적으로 다루는 힘과 연결됩니다.
