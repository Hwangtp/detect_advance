# 05 서브쿼리

## 개요

이 문서는 쿼리 안에 또 다른 쿼리를 넣는 서브쿼리(Subquery)를 다룹니다.

## 서브쿼리란?

서브쿼리는 하나의 SQL 문 안에 포함되는 또 다른 SQL 문입니다.

예를 들어 평균 점수를 먼저 구한 뒤, 그 평균보다 높은 학생만 다시 찾을 수 있습니다.

## WHERE 절의 서브쿼리

```sql
SELECT * FROM students WHERE age > (SELECT AVG(age) FROM students);
```

먼저 평균 나이를 구하고, 그 결과를 바깥쪽 쿼리의 조건으로 사용합니다.

## SELECT와 FROM 안의 서브쿼리

서브쿼리는 `SELECT`, `FROM`, `WHERE` 등 여러 위치에 들어갈 수 있습니다.

`FROM` 안에 서브쿼리를 넣으면 임시 테이블처럼 사용할 수 있습니다.

## EXISTS

`EXISTS`는 특정 조건을 만족하는 행이 존재하는지 확인할 때 사용합니다.

```sql
SELECT name FROM students s WHERE EXISTS (
    SELECT 1 FROM grades g WHERE g.student_id = s.student_id AND g.score > 95
);
```

## 서브쿼리가 유용한 상황

- 집계 결과를 조건으로 사용할 때
- 먼저 찾은 값을 다시 다른 조회에 활용할 때
- 중간 결과를 임시 테이블처럼 사용할 때

## 핵심 정리

- 서브쿼리는 쿼리 안의 쿼리입니다.
- 집계 함수와 함께 자주 사용됩니다.
- `EXISTS`는 존재 여부 확인에 적합합니다.
