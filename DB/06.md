# 06 인덱스와 뷰

## 개요

이 문서는 조회 성능과 쿼리 재사용에 관련된 인덱스(Index)와 뷰(View)를 다룹니다.

## 인덱스란?

인덱스는 데이터를 더 빠르게 찾기 위한 구조입니다.

예를 들어 이름으로 자주 검색한다면 해당 컬럼에 인덱스를 만들 수 있습니다.

```sql
CREATE INDEX idx_student_name ON students (name);
```

## 복합 인덱스

여러 컬럼을 함께 묶어 인덱스를 만들 수도 있습니다.

```sql
CREATE INDEX idx_grade_student_subject ON grades (student_id, subject_id);
```

이 방식은 두 컬럼을 함께 조건에 사용할 때 도움이 됩니다.

## SHOW INDEX

인덱스가 어떻게 생성되었는지 확인할 때 사용합니다.

```sql
SHOW INDEX FROM students;
```

## 뷰란?

뷰(View)는 SELECT 결과를 가상의 테이블처럼 저장해 두는 개념입니다.

```sql
CREATE VIEW student_grades_view AS
SELECT s.name, sub.subject_name, g.score
FROM students s
JOIN grades g ON s.student_id = g.student_id
JOIN subjects sub ON g.subject_id = sub.subject_id;
```

이후에는 뷰를 일반 테이블처럼 조회할 수 있습니다.

```sql
SELECT * FROM student_grades_view;
```

## ALTER VIEW

기존 뷰 정의를 바꿀 수 있습니다.

## 핵심 정리

- 인덱스는 검색 성능 향상에 사용됩니다.
- 뷰는 복잡한 SELECT 결과를 재사용하기 쉽게 만듭니다.
- 인덱스가 많다고 항상 좋은 것은 아니며, 필요한 컬럼에만 사용해야 합니다.
