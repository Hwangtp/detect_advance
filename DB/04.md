# 04 조인

## 개요

이 문서는 여러 테이블의 데이터를 연결해서 조회하는 JOIN을 다룹니다.

## JOIN이 필요한 이유

관계형 데이터베이스에서는 데이터를 여러 테이블로 나누어 저장합니다.  
그래서 관련 데이터를 함께 보려면 테이블을 연결해야 합니다.

예를 들어 학생 이름, 과목 이름, 점수를 한 번에 보려면 `students`, `subjects`, `grades`를 함께 사용해야 합니다.

## INNER JOIN

`INNER JOIN`은 양쪽 테이블에서 공통으로 연결되는 데이터만 조회합니다.

```sql
SELECT s.name, sub.subject_name, g.score
FROM students s
INNER JOIN grades g ON s.student_id = g.student_id
INNER JOIN subjects sub ON g.subject_id = sub.subject_id;
```

## LEFT JOIN

`LEFT JOIN`은 왼쪽 테이블의 모든 데이터를 유지하고, 연결되는 오른쪽 데이터가 있으면 함께 표시합니다.

## RIGHT JOIN

`RIGHT JOIN`은 오른쪽 테이블의 모든 데이터를 유지하고, 왼쪽 데이터가 연결되면 함께 표시합니다.

## FULL OUTER JOIN

MySQL은 `FULL OUTER JOIN`을 직접 지원하지 않기 때문에 `UNION`을 이용해 비슷한 결과를 만들 수 있습니다.

## SELF JOIN

같은 테이블을 자기 자신과 조인하는 방식입니다.  
예를 들어 친구 관계, 상하 관계 같은 데이터를 표현할 때 사용합니다.

## CROSS JOIN

두 테이블의 모든 조합을 만듭니다.  
학생 수와 과목 수의 가능한 전체 조합을 확인할 때 사용할 수 있습니다.

## ON과 USING

- `ON`: 조인 조건을 직접 명시
- `USING`: 같은 이름의 컬럼을 기준으로 조인

## 핵심 정리

- `INNER JOIN`: 공통 데이터만
- `LEFT JOIN`: 왼쪽 기준 모두
- `RIGHT JOIN`: 오른쪽 기준 모두
- `CROSS JOIN`: 모든 조합
- `SELF JOIN`: 같은 테이블끼리 연결
