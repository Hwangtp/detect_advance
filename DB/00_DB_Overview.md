# 00 DB Overview

## 개요

이 문서는 데이터베이스의 기본 개념과 관계형 데이터베이스에서 자주 사용하는 핵심 용어를 정리합니다.

## 데이터베이스란?

데이터베이스는 데이터를 체계적으로 저장하고 관리하는 시스템입니다.  
MySQL은 관계형 데이터베이스 관리 시스템(RDBMS)에 속합니다.

## 테이블, 행, 열

- 테이블: 데이터를 저장하는 2차원 구조
- 행(Row): 하나의 데이터 레코드
- 열(Column): 데이터의 속성

## 기본키와 외래키

### 기본키

기본키(Primary Key)는 테이블에서 각 행을 유일하게 식별하는 컬럼입니다.

- NULL 값을 가질 수 없습니다.
- 중복될 수 없습니다.

### 외래키

외래키(Foreign Key)는 다른 테이블의 기본키를 참조하는 컬럼입니다.

- 테이블 사이의 관계를 표현합니다.
- 데이터 무결성을 유지하는 데 도움을 줍니다.

## 기타 핵심 용어

- 스키마: 데이터베이스 구조를 정의한 설계도
- 인덱스: 검색 속도를 높이는 자료구조
- 뷰: SELECT 결과를 테이블처럼 다루는 가상 테이블

## SQL 언어 분류

- DDL: CREATE, ALTER, DROP
- DML: SELECT, INSERT, UPDATE, DELETE
- DCL: GRANT, REVOKE
- TCL: COMMIT, ROLLBACK

## 관계형 데이터베이스의 특징

- 데이터 중복을 줄일 수 있습니다.
- 데이터 무결성을 유지하기 좋습니다.
- 구조화된 조회와 관리를 할 수 있습니다.
