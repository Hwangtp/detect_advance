# 01 데이터베이스 및 테이블 생성

## 개요

이 문서는 DDL을 이용해 데이터베이스와 테이블을 만드는 과정을 다룹니다.

## DDL이란?

DDL(Data Definition Language)은 데이터베이스 구조를 정의하는 SQL 명령어입니다.

- `CREATE`
- `ALTER`
- `DROP`

## 데이터베이스 생성

```sql
CREATE DATABASE IF NOT EXISTS school_db;
```

위 구문은 `school_db` 데이터베이스를 생성합니다.  
`IF NOT EXISTS`를 사용하면 이미 존재할 때 오류를 줄일 수 있습니다.

## 데이터베이스 선택

```sql
USE school_db;
```

이후 실행하는 쿼리의 대상 데이터베이스를 지정합니다.

## 테이블 생성

이 파일에서는 다음 세 테이블을 만듭니다.

- `students`
- `subjects`
- `grades`

### students

학생 정보를 저장합니다.

- `student_id`: 기본키, 자동 증가
- `name`: 이름, 필수 입력
- `age`: 나이
- `grade`: 학년

### subjects

과목 정보를 저장합니다.

- `subject_id`: 기본키
- `subject_name`: 과목명

### grades

학생과 과목별 점수를 저장합니다.

- `grade_id`: 기본키
- `student_id`: 학생 테이블 참조
- `subject_id`: 과목 테이블 참조
- `score`: 점수

## 외래키의 의미

`grades` 테이블의 `student_id`, `subject_id`는 각각 다른 테이블을 참조합니다.

- 학생과 성적 연결
- 과목과 성적 연결

이 구조를 통해 테이블 간 관계를 만들 수 있습니다.
