# Day 01-2 강사용 대본

## 수업 목표

이 시간의 핵심은 규칙 기반 분류와 KNN 분류를 비교하면서, 머신러닝이 "데이터를 보고 가까운 예시를 참고해 판단한다"는 감각을 학생에게 심어 주는 것입니다.

## 수업 시작 멘트

"지난 시간에는 데이터 표를 읽는 연습을 했습니다. 오늘은 한 단계 더 나아가서, 진짜로 간단한 분류기를 만들어 보겠습니다."

"다만 오늘도 중요한 건 모델 이름을 외우는 것이 아니라, 모델이 어떤 생각으로 분류하는지 이해하는 것입니다."

## 실습 흐름 설명

"먼저 규칙 기반 분류기를 봅니다. 우리가 길이가 짧으면 빙어, 무게가 크면 도미처럼 직접 규칙을 적어 넣는 방식입니다. 이건 사람이 규칙을 정하는 방법입니다."

"이제 KNN으로 넘어가면 방식이 달라집니다. KNN은 '가장 비슷한 이웃을 보고 결정하자'라는 생각을 씁니다. 즉 사람이 미리 규칙을 적는 대신, 가까운 데이터를 참고해 결정하는 것입니다."

"코드를 실행하면 규칙 기반 분류기 정확도가 먼저 나오죠. 이 값은 우리가 손으로 만든 규칙이 전체 데이터에서 얼마나 잘 맞았는지를 뜻합니다. 여기서 중요한 건 정확도가 높더라도, 사람이 계속 규칙을 손으로 관리해야 한다는 점입니다."

"그 다음 거리 계산 예시가 나옵니다. 이 부분이 오늘의 핵심입니다. 숫자로는 어렵게 보이지만 사실은 '누가 더 비슷한가'를 계산한 결과입니다. 학생들에게는 'KNN은 가장 가까운 친구 의견을 듣는 모델'이라고 설명하면 잘 들어옵니다."

## 출력 해석 멘트

"이제 샘플 예측 결과가 나오죠. 예를 들어 첫 번째 샘플은 도미, 두 번째는 빙어, 세 번째는 농어처럼 예측됩니다. 이건 모델이 각 샘플과 가장 가까운 이웃들을 참고해서 낸 답입니다."

"전체 데이터 정확도가 1.0으로 나오는 부분은 학생들이 좋아할 장면입니다. 다만 이때 강사는 꼭 말해 줘야 합니다. '전체 데이터를 다 보고 전체 데이터를 다시 맞히는 것은 아직 공정한 평가가 아니다.' 이 말이 다음 시간 train/test split으로 이어지는 연결점입니다."

"최근접 이웃 정보가 나오면, 이 숫자는 모델이 어떤 샘플들을 근거로 삼았는지 보여 주는 자료라고 설명하면 됩니다. 즉 KNN은 결과만 내는 블랙박스가 아니라, 가까운 데이터를 실제로 참고하고 있다는 점을 보여 줍니다."

## 마무리 멘트

"오늘은 사람이 직접 규칙을 적는 방식과, 데이터의 가까운 이웃을 참고하는 KNN 방식을 비교했습니다. 다음 시간에는 '그런데 이렇게 전체 데이터를 다 보고 평가해도 되나?'라는 질문을 가지고 train/test 분리로 넘어가겠습니다."
