"학생들이 가장 흥미를 느끼는 지점도 바로 여기입니다. 내가 정한 물체를 실제로 찾게 만들 수 있기 때문입니다."

---

## 2. Roboflow 프로젝트 흐름 설명

강사 멘트:

"먼저 어떤 물체를 찾을지 정합니다. 예를 들어 헬멧, 안전조끼, 특정 장난감, 특정 공구처럼 수업 목적에 맞는 클래스를 정합니다. 그다음 이미지를 모으고 Roboflow에 올린 뒤, 박스를 그리고 이름을 붙입니다."

"이 과정에서 가장 중요한 것은 라벨링 품질입니다. 모델 성능은 생각보다 학습 코드보다 라벨링 품질에 더 크게 흔들릴 수 있습니다. 박스가 들쭉날쭉하면 모델도 흔들립니다."

---

## 3. data.yaml 설명

강사 멘트:

"YOLO 학습에서 `data.yaml`은 프로젝트 소개서 같은 역할을 합니다. train, val, test 경로가 어디인지, 클래스가 몇 개인지, 이름이 무엇인지가 모두 들어 있습니다."

"학생들이 이 파일을 읽을 수 있어야 나중에 프레임워크가 바뀌어도 데이터 구조를 이해할 수 있습니다. 즉 단순 암기가 아니라 구조 이해가 중요합니다."

---

## 4. CLI와 Python 학습 코드 설명

강사 멘트:

"CLI 방식은 가장 빠른 실행 방법입니다. 코랩이나 터미널에서 바로 돌리기 좋습니다. Python 방식은 노트북이나 스크립트 안에서 더 세밀하게 제어하기 좋습니다."

"둘은 방식만 다르고 본질은 같습니다. 결국 같은 data.yaml과 같은 모델 가중치를 사용해 학습을 시작하는 것입니다."

---

## 5. 하이퍼파라미터 설명

강사 멘트:

"여기서 학생들이 꼭 알아야 할 하이퍼파라미터가 있습니다. `model`, `epochs`, `imgsz`, `batch`입니다. `model`은 출발점, `epochs`는 반복 횟수, `imgsz`는 입력 크기, `batch`는 한 번에 학습하는 이미지 수입니다."

"특히 객체탐지에서는 imgsz와 batch가 GPU 상황과 바로 연결되기 때문에, 이 값을 무작정 크게 잡는 것이 아니라 환경에 맞춰 조절해야 한다는 점도 함께 말해 주세요."

---
